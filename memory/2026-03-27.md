## Property Manager V1 hardening pass
- Reviewed and summarized a hardening pass covering Postgres cutover, auth secret validation, storage abstraction, and private attachment access.
- Key implemented changes:
  - Prisma provider changed from sqlite to postgresql.
  - `AUTH_SECRET` now throws at request time in production if missing, too short, or using known demo placeholders.
  - Added storage adapter supporting local uploads and Cloudflare R2 via S3-compatible API.
  - Replaced direct filesystem attachment writes with storage adapter writes.
  - Added authorized attachment proxy route at `/api/attachments/[id]` with role-based access checks for operator, tenant, and vendor.
  - Updated operator, tenant, and vendor UI attachment rendering to use attachment IDs routed through the proxy instead of public storage paths.
  - Added `@aws-sdk/client-s3` and expanded `.env.example` for production deployment.
- Validation status reported:
  - Attachment tests passed.
  - Other test suites passed.
  - Remaining typecheck issue was described as pre-existing and test-related.

- Takeaway: this was a meaningful private-beta hardening step, with the authorized attachment proxy as the most important security improvement.
