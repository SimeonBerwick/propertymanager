# KANBAN_STRUCTURE.md

## Board Columns
- **Backlog** — ideas and approved work not started yet
- **In Progress** — actively being worked
- **Blocked** — waiting on input, dependency, or decision
- **Complete** — done and verified

## Card Metadata
Each card should include, when possible:
- Title
- Description
- Owner
- Role lens (Mario / Elon / Warren / Barry Bot / Sim)
- Priority (low / medium / high)
- Deadline or review date
- Blocker, if any

## Suggested Swimlane Logic
Use the role lens as the organizing tag:
- **Mario** — strategy, partnerships, market, growth, vision
- **Elon** — product, engineering, automation, infrastructure
- **Warren** — budget, pricing, runway, unit economics, costs
- **Barry Bot** — ops, coordination, dashboards, docs, follow-ups
- **Sim** — decisions requiring principal approval

## Priority Rules
- **High** — reduces risk, unblocks progress, or creates immediate leverage
- **Medium** — useful and valuable, but not urgent
- **Low** — optional, exploratory, or future-facing

## Board Hygiene
- Backlog should be reviewed weekly
- In Progress should stay small
- Blocked items must include the blocker explicitly
- Complete should represent actually-finished work, not half-done work

## Default Intake Questions
Before adding a card, answer:
- Why does this matter?
- Which executive lens owns it?
- What does done look like?
- Is it blocked by strategy, technical work, money, or approval?

## Example Card Format
- **Title:** Add recurring security review
- **Owner:** Barry Bot
- **Role lens:** Warren
- **Priority:** Medium
- **Description:** Create a monthly review of security posture and operational risk.
- **Done means:** Review exists, is scheduled, and results are visible.
