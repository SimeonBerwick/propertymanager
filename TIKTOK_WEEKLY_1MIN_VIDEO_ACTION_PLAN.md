# TikTok Weekly 1-Minute Video Action Plan

## Objective
Build a sustainable weekly TikTok publishing system that produces **1 strong 45–60 second video every week** without turning content into a second full-time job.

This plan is optimized for:
- consistency over volume
- simple production over fancy production
- clear positioning over random posting
- reusable workflow over one-off effort

---

## 1) Strategic Goal
The goal is **not** to "go viral every week." The goal is to build a repeatable machine that:
- sharpens Sim’s point of view
- creates audience trust through consistency
- turns ideas from daily work into short-form content
- compounds into a library of useful clips

### Success definition for the first 12 weeks
- Publish **12 consecutive weekly videos**
- Each video has:
  - one clear idea
  - one clear hook
  - one clear takeaway
  - one simple CTA
- Build a content backlog of at least **8–12 future ideas**
- Reduce weekly production time to **90 minutes or less**

### What not to optimize for initially
- expensive gear
- daily posting
- complicated edits
- trend-chasing that doesn’t fit the brand
- perfectionism

---

## 2) Positioning: What the Channel Should Feel Like
The channel should feel like:
- smart
- practical
- direct
- useful in under a minute
- less "creator theater," more "operator sharing what works"

### Recommended brand angle
Use a **builder/operator lens**:
- what Sim is learning
- what Sim is building
- what works / what doesn’t
- practical systems, tools, and observations
- decisions, tradeoffs, workflows, lessons

That angle is durable because it creates endless content from actual work instead of forcing fake inspiration.

## 3) Core Content System
Use a **3-pillar system** so content stays varied but still coherent.

## Pillar 1: Build in Public
Short updates on what Sim is working on.

Examples:
- "The app idea I’d actually build first"
- "Why I killed a feature before writing code"
- "What I learned from shipping version one too slowly"
- "How I decide whether an idea is real or just interesting"

## Pillar 2: Systems and Workflow
Short practical videos about productivity, operating systems, automation, creator systems, or execution habits.

Examples:
- "The 3-step system I use to not lose good ideas"
- "How I turn one note into a week of output"
- "My simple weekly planning system"
- "Why most workflows fail by Wednesday"

## Pillar 3: Lessons / Opinions
Clear takes, contrarian observations, and concise lessons.

Examples:
- "Most people don’t need more ideas; they need a filter"
- "Why consistency beats intensity for creators"
- "You don’t need motivation; you need a smaller scope"
- "The fastest way to kill momentum on a project"

### Content ratio
For every 4 videos:
- 2 = Systems / Workflow
- 1 = Build in Public
- 1 = Lessons / Opinions

This keeps the feed practical while still personal.

## 4) Video Format Template
Every 1-minute video should follow the same basic structure.

## Default structure
1. **Hook (0–3 sec)**
   - strong first sentence
   - no greetings
   - no setup fluff
2. **Problem / tension (3–10 sec)**
   - name the pain, mistake, or question
3. **Core insight (10–35 sec)**
   - one idea only
4. **Practical example or framework (35–50 sec)**
   - make it tangible
5. **Close + CTA (50–60 sec)**
   - simple next step

## Hook formula options
- "If you’re trying to post consistently, do this instead..."
- "The reason most people quit content after 2 weeks is simple..."
- "I stopped doing X, and making videos got much easier."
- "Here’s the workflow that makes weekly content sustainable."
- "Most people are solving the wrong problem when they start posting."

## CTA options
Keep CTAs low-friction:
- "Follow for more practical systems like this."
- "If you want, I’ll break this workflow down in another video."
- "Comment ‘system’ and I’ll make part two."
- "Save this if you’re trying to post consistently."

## 6) Weekly Production Cadence
This has to be light enough to survive real life.

## Recommended weekly cadence
### Monday — Idea selection and scripting (20–30 min)
- pick **one** topic from backlog
- write a rough hook
- write 3 bullet points
- write CTA
- do not over-script unless needed

### Wednesday — Record (20–30 min)
- record 3–5 takes
- keep setup identical each week
- choose best take immediately

### Thursday — Edit and package (20–30 min)
- trim dead space
- add captions
- add headline text
- export vertical version

### Friday — Publish and review (10–15 min)
- post video
- write simple caption
- log what was posted
- note what worked / didn’t

### Total weekly time
**~70–105 minutes**

That is the right target. If the workflow takes much longer, the system needs simplification.

## 7) Workflow: From Idea to Post
Use this exact workflow.

## Stage 1: Capture
Keep a running idea list in one markdown file or note.

For each idea, store:
- title / topic
- audience pain point
- one-sentence takeaway
- content pillar
- status: backlog / scripting / recorded / posted

## Stage 2: Select
Each week choose based on:
- relevance
- ease of recording
- clarity of message
- variety relative to recent posts

Don’t choose the most ambitious idea. Choose the clearest one.

## Stage 3: Script lightly
Use a mini script:
- Hook:
- Problem:
- 3 bullet points:
- Close / CTA:

Aim for **spoken language**, not essay language.

## Stage 4: Record fast
Rules:
- standing or seated in same location each week
- camera at eye level
- record in vertical format
- do several takes back-to-back
- don’t re-light, reframe, or re-style endlessly

## Stage 5: Edit simply
Editing checklist:
- cut pause at start/end
- cut obvious mistakes
- add captions
- add one title line
- optional jump cuts only if needed for pace
- optional B-roll only if it adds clarity

## Stage 6: Publish and learn
After posting, log:
- topic
- hook used
- watchability impression
- comments / reactions worth noting
- whether topic deserves a follow-up

## 8) Sustainable Content Backlog System
The channel lives or dies by backlog quality.

## Backlog categories
Maintain at least 3 buckets:
- **Evergreen** — timeless advice or systems
- **Current** — something Sim is building or learning now
- **Fast takes** — quick opinions or lessons

## Minimum healthy backlog
At all times, keep:
- 8+ raw ideas
- 3 script-ready ideas
- 1 emergency low-effort topic

## Emergency topic examples
These are for busy weeks:
- "One thing I stopped doing to make content easier"
- "A mistake I made this week"
- "The simplest version of my workflow"
- "What I’d do if I had to restart from zero"

If a week gets chaotic, post the emergency topic instead of missing the week.

---

## 9) Recommended Production Style
The most sustainable style is:
- direct-to-camera
- talking head
- tight framing
- clean background
- captions on screen
- minimal B-roll

### Why this style wins
- fastest to produce
- easiest to repeat
- strongest for personality and trust
- doesn’t depend on location or elaborate setup
- easiest to batch if needed

### Optional variations
Use occasionally, not as the default:
- screen recording + voiceover
- whiteboard / note breakdown
- desk shot + overhead notes
- stitched response to a comment

Default should remain simple talking-head videos.

## 10) Equipment Stack: Keep It Simple
No gear rabbit hole.

## Minimum viable stack
- **Phone camera** with vertical recording
- **Window light** or one basic LED light
- **Tripod** or phone stand
- **Lapel mic** or simple USB/phone mic if audio is weak
- quiet corner with consistent background

## Good enough setup
### Video
- modern smartphone
- 1080p vertical is enough

### Audio
Audio matters more than camera upgrades.
Use:
- wired lav mic, or
- simple wireless lav if already available

### Lighting
Best simple option:
- face a window

Backup option:
- one soft key light at 45 degrees

### Background
Use one repeatable setup:
- tidy desk
- bookshelf
- neutral wall
- one visually interesting but uncluttered element

Consistency beats novelty.

---

## 11) Software Stack: Keep Friction Low
Use the fewest tools possible.

## Recommended stack
- **Notes / scripting:** markdown or one simple notes app
- **Recording:** phone camera
- **Editing:** one mobile or desktop editor only
- **Captions:** built into editor if possible
- **Tracking:** one markdown tracker or spreadsheet

## Rule
Do not use multiple editors, multiple note systems, or complicated asset libraries unless volume increases later.

The right software stack is the one Sim will still use on a tired Thursday.

## 13) Make It Sustainable: Non-Negotiables
These rules protect the system from collapse.

## Rule 1: One video, one idea
Trying to cram 4 ideas into 60 seconds kills clarity.

## Rule 2: Consistency beats polish
A solid B+ video posted weekly is better than one "perfect" video every six weeks.

## Rule 3: Build series, not isolated posts
When a topic works, turn it into a repeatable format.

Examples:
- "1-minute system"
- "What I’m learning building X"
- "Mistake of the week"
- "How I’d do it differently"

## Rule 4: Lower activation energy
Everything should be ready:
- same recording spot
- same tripod spot
- same framing
- same editor
- same export settings

## Rule 5: Batch only when convenient
Do not force giant batch days if they create dread.
A light weekly rhythm is usually better.

## Rule 6: Measure energy, not just views
If the process becomes annoying, it will break.
Track what feels easy to make. Sustainable formats matter most.

## 14) Metrics That Actually Matter Early
Ignore vanity metrics as the main score.

Track these instead:
- Did we post this week? (yes/no)
- How long did production take?
- Was the idea clear?
- Would this topic deserve a sequel?
- Did comments reveal audience interest?
- Which hooks felt strongest?

### Simple success scorecard per video
Rate 1–5:
- Clarity
- Speed to produce
- Confidence on camera
- Audience relevance
- Follow-up potential

A video with moderate views but high repeatability is a strategic win.
